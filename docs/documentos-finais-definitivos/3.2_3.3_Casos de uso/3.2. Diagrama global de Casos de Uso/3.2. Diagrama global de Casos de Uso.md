## Visão Geral

Este diagrama apresenta a estrutura global dos casos de uso do sistema Simple Health, organizado em três módulos principais que cobrem as necessidades de cadastro, agendamento e controle de estoque.

## Atores do Sistema

### Hierarquia de Atores

**Administrativo** (ator abstrato)
- Secretária
- Gestor

**Profissional de Saúde** (ator abstrato)
- Médico

**Sistema** (ator automatizado)

## Módulos e Casos de Uso

### Cadastro e Administração

- **UC01**: Cadastrar Novo Paciente
- **UC08**: Consultar Histórico do Paciente

### Gestão de Agendamento

- **UC02**: Agendar Consulta
- **UC03**: Solicitar Encaixe *(extends UC02)*
- **UC04**: Registrar Bloqueio de Agenda
- **UC09**: Cancelar Agendamento

### Estoque e Suprimentos

- **UC05**: Dar Baixa em Insumos
- **UC06**: Processar Entrada de NF/Itens
- **UC07**: Gerar Alerta de Estoque Crítico
- **UC10**: Controlar Validade de Itens

## Código PlantUML

```plantuml
@startuml
left to right direction

' --- ATORES ---
actor "Administrativo" as Admin_Abstract
actor "Secretária" as Sec
actor "Gestor" as Ges

actor "Profissional de Saúde" as Profissional_Abstract
actor "Médico" as Med

actor "Sistema" as Sys

' --- HIERARQUIA DE HERANÇA ---
' Nível 1: Administrativo (Base para controle e gerenciamento)
Admin_Abstract <|-- Sec
Admin_Abstract <|-- Ges

' Nível 2: Profissional de Saúde (Base para quem presta serviço)
Profissional_Abstract <|-- Med

' ===============================================
' SUBSISTEMAS
' ===============================================
package "1. Cadastro e Administração" {
  usecase "Cadastrar Novo Paciente" as UC1
  usecase "Consultar Histórico do Paciente" as UC8
  usecase "Gerar Alerta de Estoque Crítico" as UC7
}

package "2. Gestão de Agendamento" {
  usecase "Agendar Consulta" as UC2
  usecase "Solicitar Encaixe" as UC3
  usecase "Registrar Bloqueio de Agenda" as UC4
  usecase "Cancelar Agendamento" as UC9
}

package "3. Estoque e Suprimentos" {
  usecase "Dar Baixa em Insumos" as UC5
  usecase "Processar Entrada de NF/Itens" as UC6
  usecase "Controlar Validade de Itens" as UC10
}

' ===============================================
' RELAÇÕES COM ATORES ABSTRATOS
' ===============================================

' --- Administrativo (Cadastro, Finanças, Controle) ---
Admin_Abstract --> UC1 : Cadastro (Registro de Paciente)
Admin_Abstract --> UC6 : Controle de Suprimentos (NF/Itens)
Admin_Abstract --> UC10 : Controle de Qualidade/Validade
Admin_Abstract --> UC9 : Gerenciamento de Fluxo

' --- Profissional de Saúde (Clínico e Agendamento Flexível) ---
Profissional_Abstract --> UC5 : Consumo Clínico
Profissional_Abstract --> UC3 : Flexibilização de Agenda
Profissional_Abstract --> UC8 : Consulta Clínica

' ===============================================
' RELAÇÕES ESPECÍFICAS
' ===============================================

' Secretária (Ações de Agendamento Primário)
Sec --> UC2

' Médico (Ações de Bloqueio de Agenda)
Med --> UC4

' Gestor (Ações de Controle de Agenda)
Ges --> UC4

' Sistema (Tarefas automáticas)
Sys --> UC7
Sys --> UC10 : <<inclui>> (Verificação)


' ===============================================
' RELAÇÕES ENTRE CASOS DE USO
' ===============================================
UC3 ..> UC2 : <<extends>>
UC5 ..> UC10 : <<uses>>
UC7 ..> UC6 : <<uses>>
@enduml
```
